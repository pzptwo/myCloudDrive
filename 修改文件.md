## Assistant
这是 Qt5 新版 `connect` 语法的一个常见"陷阱"——你把 `recvMsg()` 放在 `public:` 下面而不是 `public slots:` 下面，但它照样能运行。原因如下：

## 新版 vs 旧版 connect 的区别

你的代码使用的是 **新版 connect 语法**（函数指针方式）：
```cpp
connect(this, &myTcpSocket::readyRead, this, &myTcpSocket::recvMsg);
```

而旧版语法是（字符串宏方式）：
```cpp
connect(this, SIGNAL(readyRead()), this, SLOT(recvMsg()));
```

### 为什么旧版必须放在 `slots:` 下？

旧版 `SIGNAL()` / `SLOT()` 宏依赖 **运行时字符串匹配**，Qt 的 moc（元对象编译器）只处理 `signals:` 和 `slots:` 段中的函数，把它们注册到元对象系统中。如果 `recvMsg()` 不在 `slots:` 下面，moc 不会为它生成元对象信息，运行时就无法通过字符串找到它，连接会失败。

### 为什么新版不需要？

新版 `connect` 使用的是 **函数指针 + 编译期类型检查**。它的工作原理是对函数指针做模板推导，直接校验参数类型是否匹配，不再依赖 moc 的运行时字符串查找。只要满足两个条件就能工作：

1. **类必须包含 `Q_OBJECT` 宏** —— 你的类有
2. **函数的参数签名和信号兼容** —— `readyRead()` 无参数，`recvMsg()` 也无参数，兼容

所以 `recvMsg()` 放在 `public:` 还是 `public slots:` 对新版 `connect` 来说**没有区别**，连接都能成功建立。

## 总结

| | 旧版 `SIGNAL/SLOT` | 新版 `&Class::method` |
|---|---|---|
| 必须放在 `slots:` 下 | ✅ 是 | ❌ 不必 |
| 检查时机 | 运行时 | 编译期 |
| `Q_OBJECT` 必须 | ✅ | ✅ |

你把它挪到 `public slots:` 是正确的做法，因为这是 Qt 的编码规范，代码更清晰易读。但它能跑并不是 bug，而是 Qt5 新版 connect 的强大之处。

## 崩溃原因分析

崩溃的**根本原因**是 `deletesocket()` 里的 `delete *iter`。我们来看调用链：

```
客户端断开
  → QTcpSocket 发出 disconnected 信号
    → myTcpSocket::solvediscoonnet() 被调用
      → opedb::getInstance().handleoffline(...)  // 更新数据库
      → emit clientOffline(this)                 // 发出信号
        → mytcpServer::deletesocket(this)        // 槽函数被调用
          → delete *iter   ← 在这里把 this 即 socket 删掉了！
```

问题在于：`delete` 执行完后，**控制权还要回到 `solvediscoonnet()`**（emit 之后函数才结束），但此时 `this` 已经被释放了，这就产生了**野指针 / use-after-free**，导致未定义行为，严重时直接闪退。

Qt 文档对 `QAbstractSocket::disconnected` 信号有一个重要警告：

> **Warning:** If you need to delete the sender() of this signal in a slot connected to it, use the `deleteLater()` function.

说的就是这个场景。

### 第一步：修改 `deletesocket()` 

把 `delete *iter` 改成 `mysocket->deleteLater()`。

`deleteLater()` 的作用是：**不会立即删除对象**，而是等当前事件处理完、控制权回到事件循环后再安全删除。这样就避免了在 socket 自己的成员函数里把 `this` 删掉的问题。

**改动位置**：[mytcpserver.cpp](file:///c:/Users/bob/Desktop/myCloudDrive/review_cloudDrive/TcpServer/mytcpserver.cpp) 中的 `deletesocket()` 函数：

```cpp
void mytcpServer::deletesocket(myTcpSocket *mysocket)
{
    QList<myTcpSocket *>::iterator iter=tcpSocketList_.begin();
    for(;iter!=tcpSocketList_.end();iter++)
    {
        if(mysocket==*iter)
        {
            // 改这里：
            // 原来: delete *iter;
            //        *iter=NULL;
            // 改成:
            mysocket->deleteLater();     // 延迟删除，等事件循环处理
            tcpSocketList_.erase(iter);  // 只从链表移除
            break;
        }
    }
    for(int i=0;i<tcpSocketList_.size();i++)
    {
        qDebug()<<tcpSocketList_.at(i)->getstrName();
    }
}
```

关键变化：
- `delete *iter` → `mysocket->deleteLater()`（安全删除）
- 去掉 `*iter=NULL`（不需要了，反正马上 erase）
- `tcpSocketList_.erase(iter)` 保持不变（从链表移除）

// 第一步：创建左侧导航列表
pListW_ = new QListWidget(this);
pListW_->addItem("好友");   // 第 0 项
pListW_->addItem("图书");   // 第 1 项

// 第二步：创建两个页面
pFriend_ = new FriendLW;    // 好友页面
pBook_   = new Book;         // 图书页面

// 第三步：创建 QStackedWidget，把两个页面装进去
// QStackedWidget 像一叠卡片，同一时间只显示一张
pSW_ = new QStackedWidget;
pSW_->addWidget(pFriend_);  // 索引 0 → 好友页
pSW_->addWidget(pBook_);    // 索引 1 → 图书页

// 第四步：用 QHBoxLayout（水平布局）把左侧列表和右侧卡片叠放一起
QHBoxLayout *pMain = new QHBoxLayout;
pMain->addWidget(pListW_);  // 左侧
pMain->addWidget(pSW_);     // 右侧
setLayout(pMain);

// 第五步：信号槽 — 点击左侧某项，右侧切换到对应页面
// currentRowChanged(int) 信号连到 setCurrentIndex(int) 槽
connect(pListW_, &QListWidget::currentRowChanged,
        pSW_,     &QStackedWidget::setCurrentIndex);

关键机制：`QListWidget::currentRowChanged(int row)` 会传出当前点击的行号，`QStackedWidget::setCurrentIndex(int)` 接收行号切换页面。比如点"好友"时 row=0，右侧显示第 0 页（FriendLW）；点"图书"时 row=1，右侧显示第 1 页（Book）。

    QVBoxLayout *pMain = new QVBoxLayout;
    pMain->addLayout(pTopHBL);    // 上部
    pMain->addLayout(pMsgHBL);    // 下部

    connect(pShowOnlineUsrPB_, &QPushButton::clicked,

void FriendLW::showOnline()
{
    if (pOnline_->isHidden())
        pOnline_->show();   // 隐藏 → 显示
    else
        pOnline_->hide();   // 显示 → 隐藏
}

**这里有一个重要的跨文件关系**：`FriendLW` 里 `new online` 创建了 online 窗口对象，并且把它嵌入到自己的布局里（`pMain->addWidget(pOnline_)`）。所以 online 窗口不是独立弹出的，而是嵌在 FriendLW 页面底部，默认隐藏。
